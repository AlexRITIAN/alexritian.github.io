## AI 动态

### 1. Anthropic 发布 Claude Fable 5 和 Mythos 5

- **原文链接**：https://www.anthropic.com/news/claude-fable-5-mythos-5
- **发生了什么**：
  - Anthropic 于 6 月 9 日发布 Claude Fable 5——Mythos 级模型经安全处理后面向公众开放
  - Fable 5 在所有基准测试达到 SOTA，软件工程/知识工作/视觉/科研领域表现卓越。任务越复杂、时间越长，领先越多
  - Mythos 5 与 Fable 5 共享底层模型但撤下部分安全限制，专为网络防御设计，通过 Project Glasswing 部署
  - 定价：$10/M 输入 + $50/M 输出 tokens——不到 Mythos Preview 一半
  - 安全护栏偏保守，约 5% 会话触发拦截；部分敏感话题会降级到 Opus 4.8 回答
- **外部评论**：
  - Ethan Mollick（One Useful Thing）：《What it feels like to work with Mythos》——描述了 Mythos 级别的实际工作体验
  - Jon Ready：《If Claude Fable stops helping you, you'll never know》——讨论了安全护栏的"静默降级"问题
  - 量子位：《Claude Mythos 5 发布！5000万行代码1天搞定》
- **我的判断**：
  - 这是 Anthropic 史上最重要发布。从 Opus 到 Fable/Mythos 不是简单迭代，而是新架构跨越
  - 双重发布策略（安全版 + 无限制版）创新性地解决了安全与能力的矛盾
  - 价格减半说明推理效率有重大突破，对企业大规模部署有实质意义
  - 社区已关注到"静默降级"问题——Fable 5 可能在你不知情的情况下拒绝帮助（如果判定你是竞争对手）。这对开发者体验是潜在隐患
- **对你的影响**：
  - 🔴 尽早通过 Claude.ai 或 API 体验 Fable 5，评估 Java/Spring Boot/PostgreSQL 代码质量
  - 注意安全护栏的静默降级行为——如果 Claude 突然"变笨"，可能不是你的问题

### 2. OpenAI：Intelligence Age 产业政策 + S-1 机密提交 + Codex 客户案例

- **原文链接**：
  - Industrial Policy: https://openai.com/index/industrial-policy-for-the-intelligence-age
  - S-1: https://openai.com/index/openai-submits-confidential-s-1
  - Nextdoor: https://openai.com/index/nextdoor
  - Notion: https://openai.com/index/notion
- **发生了什么**：
  - OpenAI 发布《Industrial Policy for the Intelligence Age》——主张政府应通过产业政策加速 AI 基础设施建设（数据中心、能源、芯片）
  - 同时披露已向 SEC 机密提交 S-1（IPO 申请），与 Anthropic 几乎同期（Anthropic 于 6月1日提交）
  - 两篇 Codex 客户案例：Nextdoor（工程师无限制构建）、Notion（Codex 解锁新能力）
- **我的判断**：
  - OpenAI 和 Anthropic 几乎同时启动 IPO 流程，2026 年下半年将迎来 AI 巨头的密集上市潮
  - Industrial Policy 白皮书是 OpenAI 的游说信号——AI 基础设施正在从"技术问题"变成"国家战略问题"
  - Codex 客户案例表明 AI 编程工具正在从"个人效率工具"变成"企业级开发平台"
- **对你的影响**：
  - 关注 OpenAI/Anthropic IPO 进程，这两家公司的 S-1 文件将揭示 AI 行业的真实财务数据
  - Codex 在企业级落地说明 AI 编程正在成为标配，Java 工程师应尽早建立 AI 辅助开发的工作流

### 3. Gemma 4 12B：端侧多模态 Agent

- **原文链接**：https://www.infoq.com/news/2026/06/google-gemma4-12b-local-coding/
- **发生了什么**：Google 发布 Gemma 4 12B，支持端侧部署、多模态 Agent 工作流，采用无编码器架构
- **我的判断**：端侧 Agent 的趋势越来越明确——从云端 API 到本地运行，隐私和延迟是关键驱动因素
